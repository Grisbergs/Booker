# Booker Design System

This document defines the visual language, UI patterns, and design rules for Booker and future applications.

---

## Purpose
- Consistency across apps
- Reusable UI standards
- Faster development
- Scalable design system

---

## Core Principles
- Clean and minimal UI
- High readability
- Consistent spacing and typography
- Component-driven design
- Light + Dark mode support

---

## Design System Modules
- Brand Guide → `BRAND_GUIDE.md`
- Color + Tokens → `TOKENS.md`
- Components → `COMPONENTS.md`
- Layout Rules → `LAYOUT.md`
- Accessibility → `ACCESSIBILITY.md`

## Theme Support
- Light mode (default)
- Dark mode (supported via CSS variables)
